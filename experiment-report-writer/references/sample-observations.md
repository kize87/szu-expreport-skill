## Known Failure Mode: Task 2 Classification

A later Task 2 classification report was generated by a script that copied a converted `/private/tmp/.../template.docx`, opened it with `python-docx`, removed paragraphs between the body anchors, inserted new paragraphs, then saved the whole document. The output still contained the cover text, but Word/WPS no longer showed the first-page/report outer frame correctly.

Treat this as a regression pattern:

- `python-docx` full-document save after `.doc` conversion is not a high-fidelity form-preservation path.
- `textutil` conversion is useful for text extraction, but not reliable enough for final school-form templates.
- Use LibreOffice to prepare and render-check a high-fidelity master `.docx`.
- If the LibreOffice-rendered master already lacks the frame, ask for a Word/WPS-saved `.docx` master and fill that instead.
